弹出框

我们扩展了一些基于 bootstrap 的 modal 组件，来模拟 alert , confirm 等动作。

包括 utils.alert() , utils.confirm() , utils.dialog() 。

提示框

需要弹出提示框 alert() ，请调用 utils.alert( message [, type ] [, complete ] ) 。

也可以传入一个以各个参数作为键名的对象 utils.alert( options ) 。

参数 类型 默认 说明
message string "" 显示的内容
type string "default" 类型： "default", "info", "success", "warning", "error"
complete function null 弹出框关闭时执行的函数
title string "提示" 弹出框标题
html boolean false message 是否为 html 内容 

utils.alert("消息字符串");
utils.alert("警告！找不到对象！", "warning", function(){ /* 弹出框关闭后执行 */ });

注意： utils.alert() 不能阻塞js线程！

包括 utils.confirm() 不能像浏览器原生 alert() 一样阻塞js线程，所以对于要在弹框确定后再执行的操作，请通过回调函数传入。

例如 utils.alert("Are you sure ?", function(){ ... }) 。

需要改变弹出框的标题，或使用带有html内容的弹出框时，请调用 utils.alert( options ) 。 


utils.alert({
	title: "新标题",
	message: '请访问 <a href="//yun.isd.com/">yun.isd.com</a>',
	html: true, // 消息主体是否为html内容
	type: "warning",
	complete: function(){}
});

确认框

弹出确认框 confirm() ，调用 utils.confirm( message [, type ] [, confirm ] [, unconfirm ] ) 。

或者传入一个键值对参数 utils.confirm( options ) 。

参数 类型 默认 说明
message string "" 显示的内容
type string "default" 类型： "default", "info", "warning", "delete"
confirm function null 在确认框中，点击"确定"按钮后，将触发该函数
unconfirm function null 没有点击"确定"按钮，而关闭确认框后，将触发该函数
title string "提示" 弹出框标题
html boolean false message 是否为 html 内容 

/* 确认框ui组件 无法阻塞js线程
   所以需要使用回调函数来模拟实现
   if (confirm("msg")) { ... } else { ... } 的效果
 */
utils.confirm("你确认做此修改？", function(){
	console.log("Yes, I do.");
}, function(){
	console.log("No.");
});

utils.confirm("你确定删除这些项吗？", "delete", function(){
	console.log("deleted");
});

提示条

弹出一个在指定时间后自动消失的提示条，调用 utils.tips( message [, type ] [, duration ] ) 。

或者传入一个键值对参数 utils.tips( options ) 。

参数 类型 默认 说明
message string "" 显示的内容
type string "info" 类型： "info", "success", "warning", "danger", "error"
duration number 3000 提示条将在 duration (毫秒)后消失
html boolean false message 是否为 html 内容 

utils.tips("操作成功", "success", 1500);

弹出窗口

通过弹出窗口，打开一个动态页面，请调用 utils.dialog( url [, data ] [, title ] [, width ] [, height ] ) 。

或者传入一个键值对参数 utils.dialog( options ) 。

参数 类型 默认 说明
url string | element | $element 页面地址（或者html元素、jQuery元素）
data object | string null 请求参数，当 data 为 object 时，将使用 POST 请求，否则使用 GET
title string "" 弹出窗口初始显示的标题
width number | string 500 窗口宽度
height number | string "auto" 窗口高度 

utils.dialog("/develop/modal", {
	someKey: "some value"
}, "一个测试窗口");

// 或（如果弹出页面的html包含在了父页面中）
utils.dialog($("#dialog-demo"));

弹出窗口页面（或元素）的 html 结构如下：


<div class="modal-dialog" id="dialog-demo" style="width:800px;"><!-- 这里定义窗口宽度 -->
	<div class="modal-content">
		<div class="modal-header">
			<h4 class="modal-title">
				<!-- 这里是窗口标题 -->
				一个测试窗口
			</h4>
			<button type="button" class="modal-close" data-dismiss="modal"><span>&times;</span></button>
		</div>
		<div class="modal-body">
			<!-- 这里是窗口页面的主体 -->
			页面主体
		</div>
		<div class="modal-footer">
			<!-- 这里通常放一些按钮 这个层不是必须的 -->
			<button type="button" class="btn btn-default" data-dismiss="modal">关闭</button>
			<button type="button" class="btn btn-primary">确定</button>
		</div>
	</div>
</div>

也可以把弹出框页面直接写在父页面中（注意设置 style="display:none"），

需要弹出时，调用 utils.dialog($("#dialog-demo"))（即向 utils.dialog 传入该元素的jQuery对象）。

带有 data-dismiss="modal" 属性的元素，点击时将自动触发窗口关闭。

需要手动触发关闭当前弹出窗口，请调用 utils.dialog.close() 。

JS输出

调用 utils.pagination() 动态输出分页内容。

参数 类型 默认 说明
selector string | element 容器（jQuery选择器）
current number 1 当前页码
pages number 1 总页数
url string "#{page}" 分页url地址（其中 {page} 将被替换为页码）
displays number 9 显示的页码按钮数 

utils.pagination({
	selector: "#pager-sample",
	current: 10,
	pages: 24,
	displays: 11,
	url: "#page={page}"
});

日期选择器

日期选择器推荐使用 bootstrap-datetimepicker 。

日期+时间


<link rel="stylesheet" type="text/css" href="./css/bootstrap-datetimepicker.css">
<script type="text/javascript" src="./js/bootstrap-datetimepicker.js"></script>

<div class="input-group input-group-sm date" style="width:160px;" id="picker-1">
	<input type="text" class="form-control input-sm">
	<span class="input-group-addon"><span class="glyphicon glyphicon-calendar"></span></span>
</div>

<script type="text/javascript">
$("#picker-1").datetimepicker({
	format: "yyyy-mm-dd hh:ii",
	autoclose: true,
	language: "zh-CN",
	todayHighlight: true
});
</script>

默认布局中并没有包含日期选择器组件的样式文件和js库，需要自己在使用的页面中引用。

css: ./css/bootstrap-datetimepicker.css , js: ./js/bootstrap-datetimepicker.js 。

更多参数选项请参考 官方Demo。

表头固定

要实现一个将表头固定的效果（表头将要滚出显示窗口外时，自动固定在窗口上方），只需按以下三步操作：

  1. 将表格头部和主体分开放置到两个指定样式类的元素中。
  2. 注意使用 <col> 元素定义宽度，以使分离的表格宽度一致。
  3. 调用一行JS代码。

Loading 层

在前台异步调用接口并更新内容时，我们常需要一朵菊花覆盖到要更新的元素上，一方面给用户以等待体验，另一方面将要更新的内容遮罩住，防止前台更新前的操作。要实现这个效果，只需两步：

  1. 插入 loading 层： var loader = utils.loader(selector) 。
  2. 在合适的时候移除它： loader.close() 。
